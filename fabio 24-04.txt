Um aplicativo é formado por uma tela e um código que pode ser java ou kotlin.
O java (MainActivity.java) usa um padrão chamado CamelCase.
Classes		- todo nome de classe começa com letra maiuscula
Variáveis	- todo nome de variáveis começa com letra minúscula
Métodos		- igual ao padrão de variáveis, mas tem parênteses no final ( )
Constantes	- todas as letras em maiúsculo. Se o nome for composto, separar com um underline.
Quando o nome da classe, variável ou método for composto, a primeira letra da palavra seguinte será maiúscula;
Exemplos:
classes: MainActivity, AppCompatActivity, MediaPlaye
variáveis: nomeUsuario, editNome, editSenha
métodos: cadastrarUsuario(), checkNome();
constantes: LENG_LONG, LENG_SHORT, MATH
Nenhum nome pode conter espaço, acentuação, cecidilha ou começar com numero ou caracter especial

Para os arquivos de tela, usamos o padrão snake_case: todas minúsculas, e quando for composta, usar um underline

Para criar um app temos que primeiro desenhar a tela (layout)
E para usar imagens em nosso projeto, temos que copiar essas imagens para a pasta drawable
Existem três tipos de medida para os elementos
wrap_content	o tamanho da caixa se ajusta ao tamanho do conteúdo
match_parent	o tamanho da caixa ocupa todo o espaço do dispositivo
fixed		o tamanho eh definido pelo programador em dp
Tamanho de margem ou de elemento eh dado por dp
Tamanho de letras eh dado por sp

glossário:
width	tamanho
height	altura
parente	pai
findViewById() procura um elemento pelo seu id no layout
Intent	intenção
listener = escutar

14/03 Constraint Layout
==================
No constraint layout precisamos pelo menos de duas ligações para manter a view na posição, vertical e horizontal.
Podemos ligar o elemento ao container pai ou a outro elemento. Para isso os elementos tem que estar identificados.
A boa pratica diz que o primeiro elemento é ligado na parte de cima com a parte de cima do parente (pai)
Se quiser centralizar, ligamos também a parte direita do elemento com a parte direita o pai e a parte esquerda do elemento com a parte esquerda do pai.
Os próximos elementos ficarão ligados na parte de cima, com a parte de baixo do elemento acima.
O android facilita colocando atalhos de digitação:

digitamos	comando			para
=========	======================	=============================================================
toptotop		toptoTop		ligar a parte de cima do elemento a parte de cima do elemento
toptobott		starttoBottom	ligar a parte de cima do elemento a parte de baixo 
startto		starttoEnd	a parte esquerda com a parte direita 
endto		endtoEnd		a parte direita com a parte direita (raro usar)
startto		starttoStart	esquerda com esquerda

Quando usamos a medida wrap_content, na largura(width) e ligamos a parte esquerda com o pai e a parte direita também com o pai, o elemento fica centralizado na tela.
Se usarmos a medida wrap_content na altura (height) e ligarmos a parte de cima com a parte de cima do pai, e a parte de baixo com a parte de baixo do pai, o elemento fica centralizado vertical e horizontalmente.
Se ao invés de centralizar quisermos "esticar" o elemento para ocupar todo espaço do celular, basta mudar a largura para match_parent e ligar a esquerda e a direita. Com match_parent ele estica ao invés de centralizar
Para desenhar uma tela normalmente usamos na maior parte do tempo três elementos:
<TextView/>	para escrever títulos ou labels
<EditText/>	para campos de entrada de dados (semelhante ao input do html)
<Buttom/>	para inserir botoes
wrap_content ajusta o tamanho da caixa com o tamanho do conteudo tanto na largura quanto na altura

package com.matheus.modularappdelivery;

import android.content.Intent;
import android.os.Bundle;
import android.view.View;
import android.widget.Button;
import android.widget.EditText;
import android.widget.TextView;
import android.widget.Toast;

import androidx.activity.EdgeToEdge;
import androidx.appcompat.app.AppCompatActivity;
import androidx.core.graphics.Insets;
import androidx.core.view.ViewCompat;
import androidx.core.view.WindowInsetsCompat;

public class LoginActivity extends AppCompatActivity {
EditText email, senha;
TextView erro, criarConta;
Button entrar;


    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        EdgeToEdge.enable(this);
        setContentView(R.layout.activity_login);

        initComponents();
        entrar.setOnClickListener(new View.OnClickListener() {
            @Override
            public void onClick(View v) {
                Toast.makeText(getApplicationContext(), "Cliquei no batao entar",Toast.LENGTH_LONG).show();
                Intent intent = new Intent(getApplicationContext(), MainActivity.class);
                startActivity(intent);
            }
        });

        ViewCompat.setOnApplyWindowInsetsListener(findViewById(R.id.main), (v, insets) -> {
            Insets systemBars = insets.getInsets(WindowInsetsCompat.Type.systemBars());
            v.setPadding(systemBars.left, systemBars.top, systemBars.right, systemBars.bottom);
            return insets;
        });
    }

    private void initComponents() {
        email = findViewById(R.id.edt_email);
        senha = findViewById(R.id.edt_senha);
        erro = findViewById(R.id.mensagem_erro);
        criarConta = findViewById(R.id.criar_conta);
        entrar = findViewById(R.id.btn_entrar);
    }
}
